MySQL

2022-04-14 / 2 min read

在MySQL中创建表时，需要考虑为字段选择哪种数据类型是最为合适的，选择了合适的数据类型，会提高数据库的使用效率。
SMALLINT:储存相对比较小的整数，例如年龄，数量，工龄和学分等。
INT:储存中等整数，例如距离。
BIGINT:储存超大整数，例如科学数据。
FLOAT:储存小的数据，例如成绩，温度和测量。
DOUBLE:储存双精度的小数据，例如科学技术。
DECIMAL:以特别高的精度存储小数数据，例如货币数额，单价和科学数据。
CHAR:存储通常包含预定义字符串值，例如国家，邮编和身份证号。
VARCHAR: 储存不同长度的字符串值，例如名字，商品名称和密码。
TEXT:储存大型文本数据，例如新闻事件，产品描述和备注。
BLOB:储存二进制数据，例如图片，声音，附件和二进制文档。
DATE:储存日期，例如生日和进货日期。
TIME:储存时间或时间间隔，例如开始/结束时间，事件提醒器。
YEAR:储存年份，例如毕业年，工作年和出生年。
ENUM:储存字符属性，只能从中选一，例如性别，布尔值。
SET:储存字符属性，可从中选择多个字符的联合，例如多项选择，业余爱好和兴趣。
